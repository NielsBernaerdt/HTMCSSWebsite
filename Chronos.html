Chronos

2D C++ Engine

Github Link:

Description:

For this school assignment, the goal was to end up with a 2D engine capable of creating a simple arcade game.
Starting from a simple project containing a basic gameloop, I gradually upgraded the project using various game programming patterns.
SDL is used for the rendering, SDL Mixer for the audio and a combination of SDL Events with XInput to manage input.

Features:

- Entity Component System:

Chronos follows a "composition over inheritance" principle. GameObjects will contain components to define its behavior.

- Game Loop:

The Game Loop consists of a ProcessInput(), Update(), Render() and a sleepduration.

- Pimpl:

The InputManager has been pimpled away

- Command Pattern:

The Command Pattern is used to bind input to actions.

- Observer Pattern:

An Observer is implemented for the gamestate.(score, player lives, ...)

- EventQueue & Threading:

The AudioManager uses a Ringbuffer as EventQueue to queue audio requests. This queue gets played/loaded on a seperate thread.

- Singleton:

Some singletons like the AudioManager, ResourceManager and SceneManager are implemented.
