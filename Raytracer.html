Graphics Prog 1

C++ Software Raytracer, Software Rasterizer & Hardware Rasterizer

Github Link:

Description:

In the course of Graphics Programming 1 we covered different 3D rendering techniques and the math that is needed to efficiently calculate the pixels for the screen.
The seen techniques cover Raytracing & Rasterization, of which we did the latter both software and hardware based.

Raytracer:

We started with raytracing. With raytracing, every pixel of the screen will be calculated by shooting a ray into the world, and calculating the closest hit with the world objects.
The sphere and plane primitive were important to get started, but we moved on to triangle meshes to be able to render more interesting objects.
To make the rendering realistic we also cover the topic of PBR (Physically Based Rendering) which creates a nice visual of the 3D primitives but also aims to be a good approximation of real life lighting effects.

Rasterizer:

The next big topic was rasterization. We wrote a (software) rasterizer that is able to render 3D triangles onto the screen.
The software rasterizer conforms as closely as possible to the DirectX11 API. Like with the raytracer project, the lighting effects uses PBR techniques.

Finally, there was a smaller project that uses the DirectX 11 API and uses the calculation power of the graphics card to make a rendering.
The final project is a combination of the software rasterizer project and the DirectX 11 API where it should be possible to switch between the two renderers without graphical differences.
